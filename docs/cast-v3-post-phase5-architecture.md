# CAST v3 Post-Phase 5 Architecture

CAST v3 is organised around five major activities:

1. Evidence
2. Analyse
3. Insights
4. Review
5. Act

This operating model separates curriculum intelligence views from module-level design support and review/action workflows.

## Primary Navigation

- Home
- Upload Curriculum
- Programme Workspace
- Programme Map
- Framework Hub
- Module Builder
- Review & Enhancement
- Data Quality
- Administration

Legacy workflows remain temporarily available where routes still exist, but new development should be CAST v3-first.

## Programme Map

Programme Map answers: what exists across the curriculum?

It remains the central programme-level visual workspace for curriculum coverage, balance, alignment and evidence maturity. It should include framework overlays such as GreenComp, LifeComp, EntreComp, DigComp, programme attributes, disciplinary frameworks and professional standards.

Assessment remains a programme map layer because assessment clustering, workload, diversity, balance and strategy are programme-level design concerns.

Modality is not a Programme Map analysis layer. Programme maps may later show aggregated modality patterns, but modality decision support belongs in Module Builder.

## Module Builder

Module Builder answers: how should this module be improved?

It is the future workspace for module-level design support:

- Modality
- UDL
- Assessment Design
- Framework Alignment

Expected future flow:

Module Descriptor -> Evidence Review -> Design Guidance -> Improvement Suggestions -> Human Decision

## Review & Enhancement

Review & Enhancement answers: what should we do next?

It is the future workspace for:

- Readiness
- SWOT
- Action Planning

These are outputs from analysis and programme-team review, not Programme Map overlays.

## Data Quality

Data Quality is a cross-cutting confidence layer. CAST preserves poor source data and surfaces quality issues rather than silently cleaning or overwriting source evidence.
